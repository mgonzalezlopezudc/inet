### 1.6 PHY Layer — Error Model

The NIST and Yans error models have been extended with HE-specific branches:

- **Per-MCS modulation mapping** correctly maps MCS 0–11 to BPSK/QPSK/QAM-16/64/256/1024 in [getHeDataSuccessRate()](src/inet/physicallayer/wireless/ieee80211/packetlevel/errormodel/Ieee80211NistErrorModel.cc).
- **LDPC SNR boost**: A flat **1.5 dB** multiplicative gain (`std::pow(10.0, 1.5 / 10.0)`) is applied to LDPC-coded data.

> [!WARNING]
> The 1.5 dB LDPC coding gain is a **reasonable simulation approximation** used in ns-3, but is not directly mandated by the standard. It varies by MCS and code rate in practice. This should be documented as a modeling choice.

**Total: 5 examples, 23 configuration variants.**

### 2.2 Strengths

- **DL OFDMA** provides 7 configurations covering different scheduler strategies, bandwidth scaling (20→80 MHz), MU-MIMO, and Multi-TID — this is the most comprehensive example.
- **HE Features** is well-designed as a feature matrix: it isolates individual HE PHY features (LDPC, PE, puncturing) and then combines them.
- **BSS Coloring** provides a clear A/B comparison (disabled vs. enabled) with two overlapping BSSes.
- **TWT** expanded from 1 to 4 configurations (Baseline, IndividualUnannounced, IndividualAnnounced, Broadcast) — a significant improvement.
- All `.ini` files follow the workspace rules: `[General]` at top, `description` on configs, units on all physical quantities, comments explain non-obvious choices.

### 2.3 Weaknesses

- **Short simulation times** (0.5–2 s). These are suitable for functional testing but insufficient for meaningful throughput/latency statistics.
- **No parameter studies.** No `repeat`, no `${variableName=...}` iterators. Cannot demonstrate capacity gains without running a campaign manually.
- **Missing `description` on some configs**: `ScheduledOnly`, `MixedUora`, `EqualRus` in `ul_ofdma/omnetpp.ini` lack descriptions.
- **No visualization configuration** beyond HE MU signal labels. No throughput visualizer, no data link visualizer, no statistics collection configuration for `opp_scavetool` post-processing.
- **Fixed MCS everywhere.** Rate adaptation is not demonstrated or even configurable. All examples use a single bitrate.
- **No fingerprint tests** attached to any example. The examples are not locked against regressions.

## 3. Code Quality

### 3.1 Comments & Documentation

**Rating: ★★★★☆ (Good)**

- Standard references are pervasive: nearly every struct, function, and constant carries an IEEE 802.11-2024 clause/table citation. Examples: every field in `Ieee80211HePhyCalculator.h`, every RU helper in `Ieee80211HeRu.h`.
- Class-level doxygen comments explain purpose, algorithms, and constraints (e.g., [HeHcf](src/inet/linklayer/ieee80211/mac/coordinationfunction/HeHcf.h#L24-L38), [computeHePpduParameters](src/inet/physicallayer/wireless/ieee80211/packetlevel/Ieee80211HePhyCalculator.h#L252-L259)).
- Inline comments explain why, not just what.
- The UORA implementation in `HeUlCoordinator` is well-documented with standard clause references (26.5.4, 26.5.4.3).

**Gaps:**
- Some anonymous helper functions in `HeHcf.cc` (e.g., `isMuEligibleDataHeader`, `hasEligibleExistingFrame`) lack any documentation.
- The MCS table static declarations in `Ieee80211HeMode.h` (700+ lines of near-identical declarations) have no per-entry comments and could benefit from code generation or a table-driven approach.

### 3.2 Programming Practices

**Rating: ★★★★☆ (Good — further improved since v1)**

**Positive:**
- Non-throwing validation API: `Ieee80211HePhyValidationResult` with `operator bool()` and descriptive `.error` strings is excellent.
- Liberal use of `ASSERT()` in the MAC layer.
- `EV_INFO` / `EV_DEBUG` logging at decision points throughout HeHcf.
- `DelayedInitializer` for the MCS table avoids static initialization order issues.
- Clean separation between scheduling interface (`IIeee80211HeDlScheduler`) and implementations.
- **HE-SIG-B magic numbers replaced** with named `constexpr` constants, each citing the governing clause/table.
- **`#define DI` macro now properly scoped** — defined and `#undef`'d within `Ieee80211HeMode.h` and `Ieee80211HeMode.cc`, eliminating namespace leakage. **(NEW fix)**
- **PHY calculator partially split** — non-trivial routines moved to [Ieee80211HePhyCalculator.cc](src/inet/physicallayer/wireless/ieee80211/packetlevel/Ieee80211HePhyCalculator.cc) (329 lines), reducing the header from 560+ to 340 lines. **(NEW fix)**
- **HeSoundingCoordinator correctly uses `ITx` interface** for frame transmission, ensuring proper channel access and post-TX backoff transitions. **(NEW fix)**

**Remaining concerns:**

- **Partial header-only PHY code.** The calculator header is now 340 lines (down from 560+), but `Ieee80211HeRu.h` (354 lines, 11 `inline` functions) and `Ieee80211HeSigCodec.h` (596 lines, 5 `inline` functions) remain fully header-based. Together these three headers still total ~1 290 lines of `inline` code.
- **HeHcf reduced but still large at 1265 lines.** Sounding, preamble puncturing, and TWT gating have been extracted, reducing it from 1461 to 1265 lines. DL scheduling context collection, UL trigger construction, triggered UL response management, and SU fallback remain. This is more manageable but still carries a "god class" tendency.
- **Anonymous namespace helpers** at file scope in `HeHcf.cc` are effectively non-testable private functions.
- **MCS table is manually enumerated** — 384+ static members typed out individually. This is maintenance-heavy and error-prone.

### 3.3 Clarity & Readability

**Rating: ★★★★☆ (Good)**

- Naming conventions are consistent with INET (PascalCase classes, camelCase methods/variables).
- Enum values use descriptive names (`HE_GI_0_8_US`, `HE_CODING_LDPC`).
- The scheduler interface is clean and self-documenting via `CandidateInfo` and `ScheduleContext` structs.
- New decomposed modules (`HeSoundingCoordinator`, `HePreamblePuncturing`, `HeTwtGating`) have clear single-purpose interfaces.

### 3.4 Reusability

**Rating: ★★★★☆ (Good)**

- The scheduler is **fully pluggable**: `IIeee80211HeDlScheduler` and `IIeee80211HeUlScheduler` interfaces with NED-configurable implementations.
- Three DL schedulers (EqualSizedRUs, BacklogBased, HoLMinDelay) and two UL schedulers (EqualSizedRUs, BacklogBased) are provided.
- The PHY calculator is a pure function — no module dependencies.
- `Ieee80211HeCapabilities` struct + `negotiateHeCapabilities()` are reusable without MAC module coupling.
- **HeSoundingCoordinator** is now a standalone `cSimpleModule` submodule, independently testable and configurable via NED. **(NEW)**
- **HePreamblePuncturing** is a self-contained validation utility with no module dependencies. **(NEW)**

### 3.5 Debugging Ease

**Rating: ★★★★★ (Excellent)**

- Rich `EV_INFO` / `EV_DEBUG` logging throughout the MAC scheduling path.
- `ASSERT()` guards at critical decision points.
- Custom Qtenv visualization for HE MU signals (`displayHeMuSignalDetails`, `displayHeMuSignalPhyFields`).
- Non-throwing validation result with diagnostic error strings in PHY calculations.
- **UORA logging** with backoff state, OCW, and RA-RU selection details. **(NEW)**
- **Statistical signals** for UORA events (`heUlRandomAccessAttempt`, `heUlRandomAccessSuccess`). **(NEW)**

## 6. Architecture Improvements Since v1

### 6.1 HeHcf Decomposition

The monolithic `HeHcf` class (previously 1461 lines) has been decomposed:

| Extracted Module | Lines | Responsibility |
|---|---|---|
| [HeSoundingCoordinator](src/inet/linklayer/ieee80211/mac/coordinationfunction/HeSoundingCoordinator.h) | 296 (.h+.cc) | NDP sounding state machine (AP-side initiation, STA-side response) |

| [HeTwtGating](src/inet/linklayer/ieee80211/mac/coordinationfunction/HeTwtGating.h) | 24 | TWT sleep/wake eligibility check |

**Net result:** HeHcf.cc reduced to **1265 lines** (-196 lines / -13.4%). The extracted components are independently testable: `Ieee80211HeSoundingCoordinator_1.test` and `Ieee80211HePreamblePuncturing_1.test` exercise them in isolation.

### 6.2 PHY Calculator Decomposition

The header-only PHY calculator has been split:

| File | Lines (before) | Lines (after) |
|---|---|---|
| `Ieee80211HePhyCalculator.h` | 560+ | **340** |
| `Ieee80211HePhyCalculator.cc` | 0 | **329** |

Non-trivial computation routines (`computeHePpduParameters`, size-counting helpers) were moved to the `.cc` file, reducing header coupling.

### 6.3 UORA / OBO Counter

UORA is now fully functional with:

| Component | Location | Mechanism |
|---|---|---|
| OBO counter state | `HeUlCoordinator.ofdmaBackoff`, `ofdmaContentionWindow` | Initialized to `intuniform(0, ocwMin)` |
| OBO decrement | `selectRandomAccessRu()` | Decrements OBO by #RA-RUs per trigger; returns selected RU index when OBO reaches 0 |
| OCW update | `reportRandomAccessResult()` | Success: `OCW = OCW_min`; Failure: `OCW = min(OCW_max, 2*OCW + 1)` |
| Configuration | NED parameters | `ocwMin` (default 7), `ocwMax` (default 31) |
| Integration | `HeHcf.cc:905` | STA-side RA-RU selection during triggered UL response |
| Feedback | `HeHcf.cc:828,1134` | Reports success/failure to `HeUlCoordinator` after BA exchange |
| Signals | `heUlRandomAccessAttempt`, `heUlRandomAccessSuccess` | Statistical recording |

## 9. Summary

The 802.11ax implementation is a **substantial, correct, and maturing** addition to INET, covering the key HE features (OFDMA, MU-MIMO, LDPC, BSS Coloring, TWT, preamble puncturing, UORA) with strong standard grounding and a clean scheduler architecture. The PHY numerology is correct. The test suite (30 unit tests, 5500+ LOC) is comprehensive.

Since the first review, several significant improvements have been made:

1. ✅ **UORA fully implemented** — OBO counter, OCW doubling/reset, RA-RU selection per Clause 26.5.4
2. ✅ **HE-SIG-B now standard-compliant** — Table 27-27 encode/decode replaces the previous simplified catalog-index approach
3. ✅ **HeHcf partially decomposed** — sounding, preamble puncturing, and TWT gating extracted into standalone modules
4. ✅ **PHY calculator split** — non-trivial routines moved from header to .cc file
5. ✅ **`DI` macro properly scoped** — `#undef` prevents namespace leakage
6. ✅ **4 new unit tests** — UORA, SIG-B encoding, preamble puncturing, sounding coordinator

**Remaining open items (in priority order):**
1. **Add HE rate adaptation** — the single most impactful missing feature.
2. **Add fingerprint tests** to lock example simulation outputs.
3. **Continue HeHcf decomposition** — extract UL trigger response and DL scheduling context.
4. **Move remaining header-only code** (`Ieee80211HeRu.h`, `Ieee80211HeSigCodec.h`) to .cc files.
5. **Document modeling assumptions** (LDPC gain) and add descriptions to UL OFDMA example configs.

# Walkthrough - HE NDP Feedback Report

This walkthrough guides you through the HE NDP Feedback Report simulation example in the INET Framework.

## Background: HE NDP Feedback Report

In dense wireless networks, the Access Point (AP) needs a low-overhead way to collect quick status updates (such as resource requests or channel state feedback) from multiple stations (STAs) simultaneously.

802.11ax introduces the **NDP Feedback Report** mechanism to address this:
1. **NFRP Trigger Frame**: The AP transmits a **Null Data Packet Feedback Report Poll (NFRP) Trigger frame** to query multiple stations.
2. **HE TB NDP Feedback PPDU**: The queried stations respond simultaneously after a Short Interframe Space (SIFS).
3. **Control-Level Response**: Instead of transmitting a full data packet, the stations respond with a very compact **HE TB NDP frame** (carrying no MAC data payload). The AP identifies each station's response based on the specific Resource Unit (RU) and subcarriers it occupies. This provides the AP with instant feedback from up to 36 stations in a single control frame exchange without any data overhead or ACK sequence.

## Network Topology and Configuration

The simulation runs in a single-BSS network (`Lan80211AxUlOfdma`) where:
- **`ap`**: The Access Point.
- **`host[0..2]`**: Three wireless stations.
- **`server`**: A wired server connected to the AP.
- **Traffic**: Uplink UDP traffic is generated by the hosts destined to the server.

The `NdpFeedbackReport` config in `omnetpp.ini` is defined as:
```ini
[Config NdpFeedbackReport]
description = "The AP sends NFRP Triggers and capable STAs respond after SIFS with simultaneous MCS-0/BCC HE-TB feedback reports; the AP sends no acknowledgment."
extends = ScheduledOnly
**.wlan[*].mib.heNdpFeedbackReport = true
**.ap.wlan[*].mac.hcf.enableNdpFeedbackReport = true
**.ap.wlan[*].mac.hcf.ulTriggerCheckInterval = 20ms
```

### Key Parameters:
1. **`heNdpFeedbackReport = true`**: AP and stations advertise support for the NDP feedback report mechanism.
2. **`enableNdpFeedbackReport = true`**: Activates the AP's generation of NFRP trigger frames.
3. **`ulTriggerCheckInterval = 20ms`**: Commands the AP to check and send trigger frames every 20ms.

---

## Running the Simulation

Execute the simulation using Cmdenv:
```sh
bin/inet -u Cmdenv -c NdpFeedbackReport examples/ieee80211ax/multi_user/ndp_feedback/omnetpp.ini
```

## Verifying Results and Interpretation

After the simulation completes, check the results using `opp_scavetool`:
```sh
# Query packetSent at the hosts (Application layer)
opp_scavetool query -l -f 'name =~ "packetSent:count" and module =~ "*.host*app*"' examples/ieee80211ax/multi_user/ndp_feedback/results/*.sca

# Query packetReceived at the server (Application layer)
opp_scavetool query -l -f 'name =~ "packetReceived:count" and module =~ "*.server.app*"' examples/ieee80211ax/multi_user/ndp_feedback/results/*.sca

# Query MAC layer transmission counts
opp_scavetool query -l -f 'name =~ "packetSentToPeer:count" and module =~ "*.mac.hcf"' examples/ieee80211ax/multi_user/ndp_feedback/results/*.sca
```

### Quantitative Summary:
- **`host[0..2].app[0] packetSent:count`**: 361 packets each (Application layer).
- **`server.app[0] packetReceived:count`**: 0.
- **`ap.wlan[0].mac.hcf packetSentToPeer:count`**: 1113.
- **`host[0..2].wlan[0].mac.hcf packetSentToPeer:count`**: 690, 720, and 728.

### Interpretation of Results:
1. **Uplink Data Starvation**:
   - The server receives **0 UDP packets** because `enableUlMuOfdma` is active on the stations, which forces them to wait for data-trigger schedules (Basic/BSRP Triggers).
   - Since the `ScheduledOnly` base configuration disables random-access and scheduled data RUs (`minRandomAccessRus = 0`, `maxMuStations = 0`), the AP sends **0 data trigger frames**. The hosts' application data packets remain queued in their MAC buffers.
2. **High MAC-Layer Transmissions**:
   - The hosts still record very high MAC-layer transmission counts (~700 frames each). These represent the **HE TB NDP feedback report frames** sent in response to the AP's frequent NFRP triggers.
   - The AP sends **79 broadcast trigger frames** (which are the NFRP triggers) and receives simultaneous feedback reports from the stations.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Scheduler/tag RU metadata -> radio medium | RU indices and allocation lists cross from MU metadata into physical-band selection. |
| Radio medium -> sub-transmission analog models | Per-RU center frequency, bandwidth, and power become the inputs to path-loss/noise calculation. |
| Same-MU sub-transmission set -> receiver listening/interference path | The medium must preserve RU isolation so one RU does not contaminate another RU's physics. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `Ieee80211RadioMedium::addTransmission()` | mitigate | Validate RU indices and fail fast when the scheduler/tag allocation set cannot be mapped cleanly to a RU band. |
| T-03-02 | Tampering | Per-RU power split | mitigate | Compute RU power from bandwidth and audit conservation against the nominal channel bandwidth so drift is observable. |
| T-03-03 | Tampering | Same-MU interference policy | mitigate | Keep sub-transmissions from the same MU PPDU non-interfering and suppress the main MU transmission physically. |
| T-03-04 | Repudiation | Missing RU-specific regression coverage | mitigate | Add explicit attenuation and noise tests so behavior changes are caught instead of inferred from broad baseline tests. |
| T-03-SC | Tampering | Package installs | accept | No external package installation is planned for this C++/OMNeT++ phase. |
</threat_model>

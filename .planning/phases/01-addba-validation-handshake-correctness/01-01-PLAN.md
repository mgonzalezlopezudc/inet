## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| EDCA pending queue -> HeDlMuTxOpFs | Packet headers, receiver addresses, and TIDs cross from queued traffic into MU scheduling. |
| BA agreement handler -> MU admission | Agreement state can be missing, inactive, stale after DELBA/timeout, or unavailable when the BA handler is null. |
| HeDlMuTxOpFs -> Ieee80211HeMuTag | Once a duplicate packet is added to the HE MU tag, PHY/radio processing treats it as an allocated MU sub-transmission. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `HeDlMuTxOpFs::buildMuContainerPacket()` | mitigate | Validate exact receiver/TID with `hasActiveOriginatorBlockAckAgreement` before queue removal and HE MU tag allocation. |
| T-01-02 | Denial of Service | Null originator BA handler | mitigate | Treat null handler as MU-ineligible; leave packet pending for SU fallback rather than dereferencing. |
| T-01-03 | Tampering | Stale BA state after DELBA/timeout | mitigate | Query the BA handler during container assembly instead of trusting earlier candidate state. |
| T-01-04 | Repudiation | Skipped ineligible packet diagnostics | mitigate | Emit `EV_WARN` with packet, receiver, TID, and reason for each skipped packet per D-07. |
| T-01-SC | Tampering | Package installs | accept | No npm/pip/cargo package installation is planned for this C++/OMNeT++ phase. |
